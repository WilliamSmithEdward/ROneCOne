# ROneCOne 1.2.0

ROneCOne 1.2.0 gives VBA a real, bounded native Task path while keeping Excel and workbook state
on the thread that owns them.

## Highlights

- Hot `Task.Run` execution for verified numeric and Boolean expression lambdas on the Windows
  thread pool
- `Task.RunOnExcel` for ordinary VBA procedures, Excel objects, COM, and workbook operations
- C#-aligned `WhenAll`, `WhenAny`, continuations, cancellation, timeouts, progress, and completion
  sources across native and cooperative tasks
- Observable execution placement through `ExecutionMode`, `WorkerThreadId`, and `CurrentThreadId`
- A generated, source-verified x64 worker kernel with non-executable bytecode and task data
- Refreshed Tasks demo and guide built around two practical calculations running together

## Release evidence

Each of three fresh Microsoft 365 Excel processes passed 425 live assertions. The 1,000-operation
native `Task.Run(...).Await` scenario completed in 0.711 seconds or less against its 1.5-second
gate. Every established delegate, filtering, ordering, and indexed-collection workload also
remained inside its release gate.

Static analysis reported zero diagnostics across the loose source project, disposable test
workbook, and all six demo workbooks. Every packaged VBA module round-tripped byte-for-byte. The
six demos executed 63 practical examples without an Office or VBE popup, and all 25 worksheets
passed visual review.

## Host contract

Native workers accept only a verified zero-argument scalar expression. They cannot invoke VBA,
Excel, COM, objects, strings, or arbitrary addresses. Unsupported work fails before submission
instead of silently moving to another execution mode.

`Task.RunOnExcel` is the explicit cooperative path for host-owned work. Everything remains local
and inside one Windows x64 Microsoft 365 Excel process; ROneCOne does not require runtime VBIDE
access, generate source, launch another Excel application, install an add-in, or transmit workbook
data.

Import `ROneCOne.cls` to begin, or open the Tasks demo to see native and Excel-owned work compose
through the same await-style surface. SHA-256 checksums for every release asset are included with
the download.
