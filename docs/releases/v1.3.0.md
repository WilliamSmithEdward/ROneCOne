# ROneCOne 1.3.0

ROneCOne 1.3.0 makes everyday code lighter and honest about execution. Plain numeric literals
now flow through every typed boundary, worksheet data moves in single bulk calls, indexed reads
are O(1), and the one execution model left is the one that was always real: cooperative tasks on
Excel's own thread.

## Highlights

- Lossless numeric widening at every typed boundary, so `list.Add 30` works without `CLng`
  wrappers; a value is admitted when it promotes without loss and narrowing still fails atomically
- A worksheet Range bridge built on single bulk `Range.Value` calls: `DataTableFromRange`,
  `LoadFromRange`, `ListFromRange`, and `ToRange`
- Array-backed element and key storage: positional reads are O(1) and a ten-thousand-element
  indexed loop drops from quadratic to linear
- `ToDisplayString` renders any deferred expression or predicate as readable pseudocode such as
  `(x.Age >= 40)` for debugging and logging
- `Task.Run` now names the cooperative scheduler; the native execution slice, its embedded
  machine code, and every parallelism claim are removed
- Rewritten documentation and tutorial-voice demo modules aimed at readers who are new to the
  library

## Release evidence

Each of three fresh Microsoft 365 Excel processes passed 444 live assertions. The 1,000-operation
cooperative `Task.Run(...).Await` scenario completed in 0.086 seconds or less against its
1.5-second gate, roughly ten times faster than the removed native path measured on the same
machine. The delegate, filtering, ordering, and indexed-collection workloads all stayed inside
their established release gates, and the collection suite now also verifies numeric widening,
indexed-access scaling, and expression display live.

Static analysis reported zero diagnostics across the loose source project, disposable test
workbook, and all six demo workbooks. Every packaged VBA module round-tripped byte-for-byte. The
six demos executed 63 practical examples without an Office or VBE popup, and all 25 worksheets
passed visual review.

## Execution contract

There is one execution model. Tasks are scheduled cooperatively on Excel's thread, keep the
workbook responsive during bounded waits, and never run on separate cores; ROneCOne makes no
parallelism claims. Everything stays local inside one Windows x64 Microsoft 365 Excel process
with no VBIDE access, no generated source, no second Excel application, no installs, and no
transmission of workbook data.

Import `ROneCOne.cls` to begin, or open any demo workbook and run its macro to see the examples
fill in live. SHA-256 checksums for every release asset are included with the download.
