# ROneCOne 1.5.0

ROneCOne 1.5.0 teaches VBA to speak JSON. A serializer and strict parser shaped like
System.Text.Json join the runtime, the downloaded web response from 1.4.0's HTTP client now
lands in navigable trees and typed DataTables, and constraint enforcement loses the last
rebuild-per-operation cliff in the data layer.

## Highlights

- `ROneCOne.Json.Serialize` and `Deserialize`: a JSON object becomes an insertion-ordered
  String-to-Variant dictionary with O(1) keyed access, an array becomes a Variant list, and
  numbers arrive as the narrowest lossless type (`Long`, `LongLong`, then `Double`); output is
  compact or two-space indented, and always locale-proof
- `DeserializeTable` lands an array of objects in a typed `DataTable`: dotted columns for
  nested objects, JSON text cells for nested arrays, database nulls for absences, numeric
  column types that widen across rows, and an optional `$.data.items`-style envelope path
- Class binding without reflection: `DeserializeInto` fills an instance you have,
  `DeserializeObjects` builds instances through a zero-argument factory delegate,
  `table.ToObjects` maps rows to instances, and `DataTableFromObjects` turns instances into a
  typed table from an explicit property-name list
- Parsing is RFC 8259 strict with position-carrying typed errors (`ROneCOne.JsonError`); the
  reader and writer mechanics are adapted, with permission, from
  [ModernJsonInVBA](https://github.com/WilliamSmithEdward/ModernJsonInVBA)
- Primary-key and unique-column enforcement is now incremental index maintenance: a
  6,000-operation constrained scenario that previously exceeded the harness's 30-second
  deadline now completes in about half a second against a new 1.5-second gate
- An eighth demo workbook walks the whole JSON surface through twelve validated examples with
  no network dependency, and the HTTP demo now parses its downloaded response into a tree and
  a table

## Release evidence

Each of three fresh Microsoft 365 Excel processes passed 548 live assertions, including the
JSON model, escape and surrogate validation, RFC strictness rejections, round-trips, table
mapping, every binding direction, and incremental constraint maintenance. All twelve benchmark
scenarios met their release gates; the two new scenarios, constrained table maintenance and the
1,000-row JSON round-trip, completed in 0.50 and 0.47 seconds (medians) against 1.5-second and
2.5-second gates.

Static analysis reported zero diagnostics across the loose source project, disposable test
workbook, and all eight demo workbooks. Every packaged VBA module round-tripped byte-for-byte.
The eight demos executed 86 practical examples without an Office or VBE popup, and all 33
worksheets passed visual review.

## Execution and network contract

Tasks remain cooperative on Excel's thread; HTTP transfers overlap inside WinHTTP's own worker
threads, and ROneCOne makes no VBA parallelism claim. The JSON parser and writer run entirely
in-process. The network is touched only by the HTTP client, and only for URLs you request: the
runtime never phones home, sends no telemetry, and requires no VBIDE trust, installs, or
references. The HTTP demo and the HTTP test contract are the only release gates that need
internet access; the JSON demo runs offline.

Import `ROneCOne.cls` to begin, or open the JSON demo to watch a document become a table.
SHA-256 checksums for every release asset are included with the download.
