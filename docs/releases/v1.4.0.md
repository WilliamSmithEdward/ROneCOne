# ROneCOne 1.4.0

ROneCOne 1.4.0 teaches VBA to call the web. An awaitable HTTP client shaped like
System.Net.Http joins the runtime, and the collection and data layers lose their last
rebuild-per-operation cliffs, so writes and positional reads are now as fast as the reads that
were already quick.

## Highlights

- `ROneCOne.HttpClient()` with `GetAsync`, `GetStringAsync`, `GetByteArrayAsync`, `PostAsync`,
  `PutAsync`, `PatchAsync`, `DeleteAsync`, and `SendAsync` (any method, standard or custom),
  each returning an awaitable Task riding in-process WinHTTP with no references or installs
- Requests overlap in flight: `Task.WhenAll` over several `GetAsync` calls waits roughly as
  long as the slowest single download, while VBA stays on one thread
- Typed failure handling at the await site with `HttpRequestException`
  (`ROneCOne.HttpRequestError`), `EnsureSuccessStatusCode`, and cancellation that aborts the
  transport
- Keyed mutation maintains the hash index in place: the tally pattern `d(k) = d(k) + 1` is O(1)
  per call, and a 12,000-operation update-and-remove scenario that previously exceeded the
  harness deadline now completes in about half a second
- Indexed list writes, `Rows` loops, and positional reads on sets, queues, and dictionaries are
  O(1) per access through lazy version-checked enumeration and version-cached snapshots
- Two shipped-behavior fixes: re-filtering an already-read DataView now refreshes instead of
  serving stale results, and `AddColumn` on a populated table backfills every existing row
- Demos and documentation drop the `CLng`/`CDbl`/`&` ceremony that lossless widening made
  unnecessary, and a seventh demo workbook downloads live data from https://pokeapi.co

## Release evidence

Each of three fresh Microsoft 365 Excel processes passed 505 live assertions, including twenty
HTTP contracts against https://pokeapi.co covering success, 404 fault paths, overlapped
downloads, cancellation, headers, and custom verbs. All ten benchmark scenarios met their
release gates; the four new mutation and positional scenarios completed in 0.47, 0.066, 0.047,
and 0.012 seconds (medians) against 1.5-second gates.

Static analysis reported zero diagnostics across the loose source project, disposable test
workbook, and all seven demo workbooks. Every packaged VBA module round-tripped byte-for-byte.
The seven demos executed 72 practical examples without an Office or VBE popup, and all 29
worksheets passed visual review.

## Execution and network contract

Tasks remain cooperative on Excel's thread; HTTP transfers overlap inside WinHTTP's own worker
threads, and ROneCOne makes no VBA parallelism claim. The network is touched only by the HTTP
client, and only for URLs you request: the runtime never phones home, sends no telemetry, and
requires no VBIDE trust, installs, or references. The HTTP demo and the HTTP test contract are
the only release gates that need internet access.

Import `ROneCOne.cls` to begin, or open the HTTP demo to watch three downloads overlap live.
SHA-256 checksums for every release asset are included with the download.
