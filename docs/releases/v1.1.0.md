# ROneCOne 1.1.0

ROneCOne 1.1.0 makes its C#-inspired surface faster, safer, and more direct while preserving the
single dependency-free VBA class module.

## Highlights

- Indexed dictionaries, hash sets, lookups, table keys, and relations with capacity control
- Direct `WhenAll(task...)`, bounded `WaitAsync`, `YieldOnce`, aggregate task failures, and
  disposable cancellation registrations
- Fluent `Column(...).AsPrimaryKey` and `Table.Row(...).Add` data construction with explicit
  `ROneCOne.DBNull`
- Deterministic `Using(resource).Run(work)` cleanup and truthful provider async capability reporting
- Transactional and continue-on-error DataAdapter update options
- Refreshed Collections, Tasks, and Data living demos using the shortest proven syntax

## Release evidence

Each of three fresh Microsoft 365 Excel processes passed 414 live assertions. The 100,000-lookup
dictionary scenario completed in 1.122 seconds or less; the 10,000-item build-and-read scenario
completed in 0.571 seconds or less. All delegate, filtering, ordering, and hash scenarios remained
inside their release gates.

Static analysis reported zero diagnostics across the loose source project, disposable test
workbook, and all six demo workbooks. Every packaged VBA module round-tripped byte-for-byte. All
six demos executed without an Office or VBE popup, and every one of their 25 worksheets passed
visual review.

## Host contract

The runtime targets Windows x64 Microsoft 365 Excel and stays inside one Excel process. Tasks and
provider operations are cooperative because Excel owns VBA and workbook COM state on one thread.
ROneCOne does not require runtime VBIDE access, generate source, install an add-in, launch another
Excel application, or transmit workbook data.

Import `ROneCOne.cls` to begin, or open a demo workbook to see each capability run in Excel.
SHA-256 checksums for every release asset are included with the download.
